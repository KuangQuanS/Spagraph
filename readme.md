# 多模态空间细胞通讯分析框架设计

本项目旨在基于空间转录组数据（HE 图像 + 基因表达）构建空间细胞通讯图谱，识别关键调控基因与通信热点区域。该框架结合自监督预训练、多模态融合、注意力机制筛选调控基因，并借助图神经网络探索空间通信网络。

---

## 模块概览

1. 自监督预训练
2. 多模态融合与调控基因筛选
3. 空间通信图构建
4. 下游分析与可视化

---

## Ⅰ. 自监督预训练（Self-supervised Pretraining，保证注意力分数有语义意义）

### 🔹 图像模态（HE 图）

- 输入：每个 spot 对应的 patch 图像
- 模型：ViT（Vision Transformer）
- 方法：使用 MAE / DINO 进行掩码自监督学习
- 输出：
  - 每个 patch 的图像 embedding
  - 每个 patch 内部注意力权重（可分析组织结构）

---

### 🔹 基因表达模态（基因名句子）

- 输入：每个 spot 的基因名序列
（按表达量排序，包含H M L等来提示表达量，不用数值避免数据过于复杂难以拟合，目前用的阈值方法，标准化和对数化后小于2的为L，大于2小于3的为M，大于3的为H）
- 模型：BERT
- 方法：BERT MLM（mask language model）预训练
- 输出：
  - 每个基因 token 的 embedding
  - 每个 gene 在该 spot 的注意力分数（用于筛选调控基因）

---

## Ⅱ. 多模态融合与调控基因筛选

### 🔸 融合机制

- 使用双向交叉注意力（Cross-Attention）机制：
  - 图像→表达注意力
  - 表达→图像注意力
- 双向注意力后加一个对比学习对齐模态（loss_fusion = contrastive_loss(z_img2gene, z_gene2img)
- 再接一个concat得到融合的embedding
- 输出：每个 spot 的融合embedding（用于图构建）

### 🔸 调控基因筛选

- 基于 BERT 的注意力权重，设定一个阈值，如果分数大于阈值则说明这个token是调控基因
- 对每个spot，对其K个邻居：
  - 设置一个通信打分
  - 高分受体/配体基因 → 视为“激活”
  - 低分/未被注意的基因 → 忽略（视为无通讯）

---

## Ⅲ. 细胞通讯图构建

### 🔸 节点构造V

- 每个 spot 作为图中一个节点
- 节点特征：多模态融合 embedding
- 额外注入：随机游走位置编码 RWPE(random walk position embedding)，让transformer具备图结构感知
对图 G 进行 k-step 随机游走，得到每个节点的访问概率分布,将 RWPE 作为 拓扑位置编码 p_spot

### 🔸 边构造E（通讯关系）

- 若两个邻居spot *i* 表达激活的配体基因，且邻近 spot *j* 表达激活的受体基因：
  - 建立边 *(i → j)*
  - 边的特征（bias）包括： attention_i × attention_j x prior（来自数据库的 LR 对置信度，这个可以当作ligand–receptor coexpression scores）, pair id , 空间距离

- 最后得到稀疏图G=(V,E)

### 🔸 GNN 模块

- 输入图：上述构建好的通讯图，包括节点特征，位置偏置，边特征
- Augment：随机删除边 & 加噪特征，构建一个增强图，另一个增强图采用随机游走子图增强，前者强调局部扰动，后者强调拓扑裁剪
- 模型：Graphormer，模型既考虑既考虑节点特征相似性，也考虑空间距离、LR 打分，不只看1-hop领域，还能捕捉全局通讯依赖
- 任务：对比学习，使用infoNCE损失，将两个编码过的特征图作为正样本对，其他节点作为负样本，让同一节点在两个视图下保持一致
- 输出：最终节点embedding，可以用于聚类、通讯模块识别，将注意力分数作为信号通讯强度

## Ⅳ. 下游分析与可视化

- 📌 热点区域识别（通讯密集子图）
- 🔁 Relay 通讯链路检测（模仿 CellNEST）
- 🧬 关键配体/受体网络统计
- 🔍 与组织结构 / marker gene 区域进行富集比对
- 📊 可视化输出：
  - 热图、力导图、投影图（UMAP/PCA）、relay 饼图等

- 可以找一些关键spot，后面把spot之间换成基因之间也可以，去找关键基因

## ✅ 技术要点回顾

| 模块 | 是否必须 | 技术说明 |
|------|----------|----------|
| 自监督预训练 | ✅ 必须 | 保证注意力分数具有语义信息 |
| KL 散度监督 | ❌ 不再使用 | 关注点已转为通讯而非分类 |
| 注意力筛选 | ✅ 必须 | 控制通讯边界构建 |
| GNN 模型 | 🔄 可选 | 用于进一步聚类、区域识别等 |
| 多次图运行集成 | 🔄 可选 | 提升通讯图稳定性（可参考 CellNEST） |
